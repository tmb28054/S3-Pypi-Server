## [0.3.0] - 2026-05-08

### Added

- Optional KMS encryption support via `EnableKMSEncryption` parameter — creates a Customer Managed Key and encrypts S3, DynamoDB, and CloudWatch Logs at rest.
- Optional API Gateway Lambda authorizer via `EnableAuthorizer` parameter supporting dual authentication methods.
- LDAP/AD authentication (Basic Auth) — validates credentials against an LDAP server configured in Secrets Manager.
- DynamoDB API key authentication using `__token__` username convention (matching PyPI standards).
- Read/write access levels on API keys (`read` for install-only, `read/write` for install + upload).
- Second LDAP entitlement group (`write_entitlement_group`) for granting upload permissions.
- Twine-compatible package upload via POST to `/simple/` endpoint with Upload Lambda.
- `apikey` CLI subcommand with `create`, `list`, `get`, `delete`, and `update` actions for managing API keys.
- `--access` flag on `apikey create` and `apikey update` for setting access levels.
- `configure-ldap` CLI subcommand for populating LDAP configuration in Secrets Manager.
- `--write-entitlement-group` flag on `configure-ldap` for write access group.
- Optional VPC support via `SubnetIds` and `VpcId` parameters — places Lambda functions in a VPC with a dedicated Security Group.
- Lambda LogGroups with 30-day retention for all Lambda functions.
- API Gateway access logging with 30-day retention including request ID, source IP, HTTP method, resource path, status, and response length.
- `--api-key-table-name` and `--ldap-secret-arn` flags on the `configure` subcommand.
- CloudFormation outputs: `KMSKeyArn`, `LDAPSecretArn`, `APIKeyTableName` (conditional).
- `s3pypi/apikey.py` module for DynamoDB API key CRUD operations with access levels.
- `s3pypi/secrets.py` module for Secrets Manager LDAP configuration updates.
- Interactive prompts when running `s3pypi configure` without flags — displays current values and accepts input for each setting.
- `s3pypi configure --from-stack <name>` to import configuration from an existing CloudFormation stack's outputs.
- `s3pypi deploy` subcommand with named flags for all template parameters (`--enable-authorizer`, `--enable-kms-encryption`, `--vpc-id`, etc.).
- `s3pypi deploy --update <name>` to update an existing stack with interactive prompts showing current parameter values.
- `s3pypi pip` subcommand to generate pip configuration with keyring credential storage.
- `s3pypi pip --save` to write pip config directly to the OS-appropriate location.
- `s3pypi twine` subcommand to generate `.pypirc` configuration with keyring credential storage.
- `s3pypi twine --save` to write `.pypirc` to the home directory.
- `keyring` added as a runtime dependency for secure credential storage.
- `CloudFrontDistributionId` and `CloudFrontURL` and `ApiGatewayURL` stack outputs.
- CloudFront now forwards POST/PUT/DELETE methods and `Authorization`/`Content-Type` headers to support twine uploads.
- Authorizer Lambda logs the generated policy for debugging.
- Authorizer accepts `write`, `read/write`, `readwrite`, and `rw` as write access levels.
- Smoke tests for cfn-lint and yamllint validation.
- yamllint rule overrides via inline comments in YAML files (no `.yamllint` config file).

### Changed

- S3 bucket name is now randomly generated by CloudFormation (no longer derived from stack name prefix and account ID). Retrieve from `BucketName` stack output.
- DynamoDB API key table name is now randomly generated by CloudFormation (no longer hardcoded). Retrieve from `APIKeyTableName` stack output.
- Replaced `deploy.sh` with `s3pypi deploy` CLI subcommand — deploys via boto3 and auto-saves stack outputs to config.
- CloudFormation template is now bundled inside the Python package (no external file needed).

### Removed

- `deploy.sh` — replaced by `s3pypi deploy` subcommand.

## [0.2.0] - 2026-05-03

### Added

- `configure` subcommand to save default `--bucket` and `--cloudfront-distribution-id` to `~/.s3pypi/config.json`.
- Config fallback: `upload` reads bucket and distribution ID from config when CLI flags are omitted.
- `s3pypi/config.py` module for configuration file management.
- Optional `DomainName` and `AcmCertificateArn` CloudFormation parameters for custom domain support.
- Conditional CloudFront alias and ACM certificate configuration via `HasCustomDomain` condition.
- `PyPIEndpoint` output now returns the custom domain when provided.
- Project URLs (Homepage, Repository) in `pyproject.toml`.

### Changed

- `--bucket` is now optional on the `upload` subcommand (falls back to configured value).

## [0.1.1] - 2026-05-03

### Fixed

- Changed build backend from internal `setuptools.backends._legacy:_Backend` to `setuptools.build_meta`.
- Added `@pytest.mark.smoke` markers to pylint and bandit smoke tests so `pytest -m smoke` works.
- Registered `smoke` marker in `pyproject.toml` to suppress unknown marker warnings.

### Added

- `pytest-cov` added to test dependencies.
- `Name`, `Environment`, and `Project` tags on all taggable CloudFormation resources.

## [0.1.0] - 2026-05-03

### Added

- PEP 503 compliant package name normalization and distribution filename parsing.
- HTML index and detail page generation with round-trip parsing support.
- S3 uploader with automatic index page regeneration after each upload.
- CloudFront cache invalidation support for immediate index freshness.
- CLI entry point (`s3pypi upload`) with `--bucket` and `--cloudfront-distribution-id` flags.
- CloudFormation template for full AWS infrastructure: S3 bucket, API Gateway, CloudFront distribution.
- Deployment script (`deploy.sh`) wrapping `aws cloudformation deploy`.
- Property-based test suite using Hypothesis (5 properties).
- Unit test suite with 100% code coverage.
- Smoke tests for pylint and bandit validation.
- README with architecture overview, quickstart, CLI usage, and development setup.
- LICENSE (MIT).
- `docs/` folder with design documentation, user guide, and FAQ.
- `.gitignore` for Python projects.
- Git repository initialized.
